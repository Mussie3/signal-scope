# SignalScope

SignalScope is a side project I'm building to get better at writing frontends that have to deal with constantly changing data. The idea is a map-based dashboard where you can watch entities move, alert, and interact with each other in real time — kind of like a stripped-down ops/monitoring view.

There's no real backend yet; data is generated by a local simulation loop. The interesting parts (for me, anyway) are everything that happens between "data changes 10 times a second" and "the UI doesn't fall over."

## What's in here so far

- A map view rendered with Leaflet
- A simulation loop that drives entity state updates
- Zustand for state, with selective subscriptions so components only re-render when their slice changes
- Light/dark theme that swaps both UI colors and map tile layers
- A top bar with search and a theme toggle (the toggle was annoying to get right — that's how this project actually started)

A bunch of things on the roadmap aren't built yet. See the bottom.

## Stack

- **React + TypeScript + Vite** — standard setup
- **Zustand** — picked it over Redux/Context because the API is small and selective subscriptions are easy
- **Tailwind CSS** — with CSS variables for theming
- **Leaflet** — lightweight, doesn't pull in a giant map SDK
- **ESLint + Prettier**

## Folder layout

```
src/
├── app/          entry + global setup
├── features/     one folder per domain
│   ├── map/        rendering
│   ├── entities/   data model + store
│   ├── simulation/ the loop that mutates state
│   ├── alerts/     events triggered by entity changes
│   └── ui/         layout components (top bar, panels)
├── shared/       things used across features (UI, utils, store)
└── styles/       globals + theme tokens
```

I'm trying to keep `features/` self-contained and `shared/` small. If something starts being imported from three features it gets promoted out.

## Running it

```bash
npm install
npm run dev
```

Production build:

```bash
npm run build
```

## Why I'm building this

Mostly to push on:

- handling lots of small state updates without trashing render performance
- structuring a React app around features instead of layers (`components/`, `hooks/`, etc.)
- working with Leaflet, which I hadn't really used before
- making the UI feel calm even when the underlying data is noisy

## Roadmap
